Codex Planner × Claude CLI Builder

AI 개발 파이프라인 흐름도

개발 요청을 Codex가 분석·계획하고, 사람이 승인한 뒤 Claude CLI가 구현한다. 계획 단계는 CLI 스크립트와 Codex 앱 두 경로로 진입할 수 있으며, 두 경로 모두 동일한 codex-plan.json으로 수렴한다.

Codex CLI — codex exec (read-only) Codex 앱 — $dev-planner 스킬 사람 — 승인 게이트 Claude CLI — Builder

실행 전제 (미충족 시 파이프라인 중단)

① git 저장소 필요 — git init 없이는 git status/diff 기반 구현·검토가 무의미. ② POSIX 셸 필요 — ai-dev.sh는 bash 스크립트로 Windows에선 Git Bash / WSL에서만 동작(PowerShell 불가). ③ CLI 로그인 — codex·claude·python PATH 등록 및 인증.

Plan 단계 · Codex

사전 준비 · 스킬 탐색

.agents/skills/dev-planner/SKILL.md는 Codex가 자동 탐색한다(cwd→repo root 스캔, config.toml 등록 불필요). AGENTS.md는 세션마다 자동 로드되어 Planner 규칙을 강제한다. 단, codex exec 비대화형에서 $dev-planner 명시 호출은 문서상 미확정 — 프롬프트에 규칙을 인라인 재기술해 폴백을 둔다(현재 스크립트가 이미 수행).

진입 A · CLI (자동/비대화형)

$ ./scripts/ai-dev.sh "…요청…"

codex exec --sandbox read-only 로 스킬 실행 → 원본 출력 → Python으로 JSON 파싱. 파싱 실패 시 원본 표시 후 중단.

→ user-prompt.txt · codex-plan.raw

진입 B · Codex 앱 (대화형)

$dev-planner …요청…

앱 composer에서 스킬을 대화형 실행 → 질문·가정을 즉석 해소. 수동 브리지 필요: 스크립트가 run 디렉터리 생성·스키마 검증을 안 하므로, 확정 JSON을 직접 codex-plan.json으로 저장해야 함(read-only면 저장 불가 → workspace-write 승인).

핸드오프 원칙

Codex(CLI·앱)는 Claude를 직접 호출하지 않는다. 빌드는 사람이 실행하는 ./scripts/ai-build.sh <run-dir> 로만 이어진다(※ 이 스크립트는 신규 작성 필요 — 원본엔 없음). 승인 전 questions를 반드시 해소할 것: claude -p는 자율 실행이라 미해결 질문을 임의 가정으로 진행한다.
